# Maestro Pack, Publish, and Deploy Guide

Detailed guide for the packaging, publishing, and deployment stages of the Maestro lifecycle.

## Overview

The deployment pipeline follows this sequence:

```
Pack Project → Add to Solution → Pack Solution → Publish → Configure → Deploy Config → Deploy
```

Each step depends on the output of the previous step. Do not skip steps.

## Pack the Maestro Project

Package the Maestro project into a `.nupkg` file:

```bash
mkdir -p <directory>/output
uip maestro pack <directory>/<solutionName>/<projectName> <directory>/output --version 1.0.0
```

**Output naming pattern:** `<projectName>.processOrchestration.ProcessOrchestration.<version>.nupkg`

### Version Management

- Default version: `1.0.0`
- If redeploying, bump the version (e.g., `1.0.1`, `1.0.2`)
- Publishing the same version twice will fail

### Required Project Files

The following files must exist at the project root (NOT in a `content/` subfolder):
- `<projectName>.bpmn` — BPMN process definition
- `entry-points.json` — Start event entry points (auto-generated by `maestro init`)
- `package-descriptor.json` — Package metadata
- `operate.json` — Runtime configuration

## Add Project to Solution

The project must already be inside the solution folder:

```bash
uip solution project add \
  <directory>/<solutionName>/<projectName> \
  <directory>/<solutionName>/<solutionName>.uipx
```

A warning about `SolutionsMetadataReader` is benign and can be ignored.

## Pack the Solution

Package the entire solution into a `.zip`:

```bash
mkdir -p <directory>/solution-output
uip solution pack <directory>/<solutionName> <directory>/solution-output
```

## Publish to Orchestrator

Upload the solution zip to Orchestrator:

```bash
uip solution publish <directory>/solution-output/<solutionZipFile> --output json
```

**Key output:** `PackageVersionKey` — required for the configuration step.

## Create a Configuration

There is no CLI command for configuration creation yet. Use a direct API call:

```bash
source ~/.uipath/.auth && \
BASE_URL="${UIPATH_URL}/${UIPATH_ORGANIZATION_ID}/${UIPATH_TENANT_NAME}/automationsolutions_" && \
curl -s -X POST "${BASE_URL}/api/configurations" \
  -H "Authorization: Bearer ${UIPATH_ACCESS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"packageVersionKey":"<PACKAGE_VERSION_KEY>","name":"<configName>"}'
```

The response is a quoted GUID string — this is the **configuration key**.

### Auth File Location

The `~/.uipath/.auth` file is created by `uip login` and contains:
- `UIPATH_URL` — Base URL
- `UIPATH_ORGANIZATION_ID` — Organization ID
- `UIPATH_TENANT_NAME` — Tenant name
- `UIPATH_ACCESS_TOKEN` — Bearer token

## Generate Deploy Configuration

Generate a default deploy configuration file:

```bash
uip solution deploy config <solutionName> --package-version <version> -d <directory>/deploy-config.json --output json
```

This creates a JSON file with default deployment settings that can be customized before deploying.

## Deploy

Deploy the solution to Orchestrator:

```bash
uip solution deploy run \
  --name "<deploymentName>" \
  --package-name "<solutionName>" \
  --package-version "<version>" \
  --folder-name "<projectName>_folder" \
  --folder-path "Shared" \
  --config-file <directory>/deploy-config.json \
  --output json
```

### Important Notes

- The `deploy run` command polls automatically — no separate polling step needed
- There is NO `--configuration-key` option; use `--config-file` instead
- The command returns a terminal status when complete

### Terminal Statuses

| Status | Meaning | Action |
|--------|---------|--------|
| `DeploymentSucceeded` | Full success | Proceed to run |
| `SuccessfulInstall` | Install succeeded | Proceed to run |
| `SuccessfulActivate` | Activation succeeded | Proceed to run |
| `FailedInstall` | Install failed | Stop and report error |
| `FailedUpgrade` | Upgrade failed | Stop and report error |

### Deployment Timeout

If deployment stays in progress for more than 5 minutes, warn the user and ask if they want to continue waiting or check manually.

## Redeployment

To redeploy with changes:

1. Bump the version number
2. Re-pack the project: `uip maestro pack ... --version <newVersion>`
3. Re-pack the solution: `uip solution pack ...`
4. Re-publish: `uip solution publish ...`
5. Create new configuration with the new `PackageVersionKey`
6. Generate new deploy config: `uip solution deploy config ... --package-version <newVersion>`
7. Deploy: `uip solution deploy run ... --package-version <newVersion>`
