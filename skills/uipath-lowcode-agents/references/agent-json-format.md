## Project Directory Structure

After running `uip lowcodeagents init <name>`, the following structure is created:

```
<AgentName>/
├── agent.json              # Main agent configuration (edit this)
├── entry-points.json       # Entry point definition (must mirror agent.json schemas)
├── project.uiproj          # Project metadata
├── flow-layout.json        # UI layout — do not edit
├── .agent-builder/         # Studio Web artifacts — do not create/edit/regenerate
│   ├── agent.json
│   ├── entry-points.json
│   └── bindings.json
├── .project/               # Build metadata — do not edit
│   └── JitCustomTypes.json
├── evals/                  # Evaluation sets and evaluators
│   ├── eval-sets/
│   │   └── evaluation-set-default.json
│   └── evaluators/
│       ├── evaluator-default.json
│       └── evaluator-default-trajectory.json
├── features/               # Context grounding / memory spaces
└── resources/              # Escalations, tools, contexts, MCPs
```

## agent.json Schema

This is the primary configuration file. Edit it directly to configure the agent.

```json
{
  "version": "1.0.0",
  "settings": {
    "model": "<MODEL_IDENTIFIER>",
    "maxTokens": 16384,
    "temperature": 0,
    "engine": "basic-v2",
    "maxIterations": 25,
    "mode": "standard"
  },
  "inputSchema": {
    "type": "object",
    "required": ["<FIELD_NAME>"],
    "properties": {
      "<FIELD_NAME>": {
        "type": "string",
        "description": "<FIELD_DESCRIPTION>"
      }
    }
  },
  "outputSchema": {
    "type": "object",
    "properties": {
      "<FIELD_NAME>": {
        "type": "string",
        "description": "<FIELD_DESCRIPTION>"
      }
    }
  },
  "metadata": {
    "storageVersion": "50.0.0",
    "isConversational": false,
    "targetRuntime": "pythonAgent",
    "showProjectCreationExperience": false
  },
  "type": "lowCode",
  "projectId": "<AUTO_GENERATED_UUID>",
  "messages": [
    {
      "role": "system",
      "content": "<SYSTEM_PROMPT_TEXT>",
      "contentTokens": []
    },
    {
      "role": "user",
      "content": "{{input.fieldName}} {{input.otherField}}",
      "contentTokens": []
    }
  ]
}
```

### Field Reference

| Field | Type | Description |
|-------|------|-------------|
| `version` | string | Agent config version. Use `"1.0.0"` for new agents. |
| `settings.model` | string | LLM model identifier (e.g., `"anthropic.claude-sonnet-4-6"`, `"gpt-4.1-2025-04-14"`). A `uip lowcodeagents models` discovery command is planned. |
| `settings.maxTokens` | number | Maximum output tokens. Common values: 16384, 32768. |
| `settings.temperature` | number | Sampling temperature (0 = deterministic, 1 = creative). Use 0 for rule-based agents. |
| `settings.engine` | string | Execution engine. Use `"basic-v2"`. |
| `settings.maxIterations` | number | Max agent loop iterations. Default 25. |
| `settings.mode` | string | Execution mode. Use `"standard"`. |
| `inputSchema` | object | JSON Schema for agent inputs. Must match `entry-points.json` input. |
| `outputSchema` | object | JSON Schema for agent outputs. Must match `entry-points.json` output. |
| `metadata.storageVersion` | string | Storage format version. Use the value generated by init. |
| `metadata.isConversational` | boolean | Always `false` for autonomous agents. |
| `metadata.targetRuntime` | string | Always `"pythonAgent"`. |
| `type` | string | Always `"lowCode"`. |
| `projectId` | string | UUID auto-generated by `uip lowcodeagents init`. Do not modify. |
| `messages` | array | System and user message templates. |

### Input/Output Schema Types

Schema properties support standard JSON Schema types:

| Type | Example | Use For |
|------|---------|---------|
| `"string"` | `"type": "string"` | Text, JSON strings, formatted data |
| `"number"` | `"type": "number"` | Numeric values with decimals |
| `"integer"` | `"type": "integer"` | Whole numbers |
| `"boolean"` | `"type": "boolean"` | True/false flags |
| `"object"` | `"type": "object"` | Nested structures |
| `"array"` | `"type": "array"` | Lists |

### Messages

Messages define the agent's behavior through system and user prompts.

**System message** — sets the agent's role, rules, and behavior:
```json
{
  "role": "system",
  "content": "You are a Risk Assessor Agent. Your role is to...",
  "contentTokens": [
    {
      "type": "simpleText",
      "rawString": "You are a Risk Assessor Agent. Your role is to..."
    }
  ]
}
```

**User message** — templates the user's input using `{{input.fieldName}}` variables:
```json
{
  "role": "user",
  "content": "{{input.documentFields}} {{input.companyEnrichment}}",
  "contentTokens": [
    { "type": "variable", "rawString": "input.documentFields" },
    { "type": "simpleText", "rawString": " " },
    { "type": "variable", "rawString": "input.companyEnrichment" }
  ]
}
```

## contentTokens Construction

Every message in `messages[]` must have both a `content` string and a matching `contentTokens` array. These must always be in sync — when editing one, update the other.

### Tokenization Rules

1. Parse the `content` string from left to right
2. Text outside `{{ }}` delimiters becomes a `simpleText` token
3. Text inside `{{ }}` delimiters becomes a `variable` token (strip the `{{` and `}}`)
4. Every segment — including whitespace between variables — gets its own token entry
5. Variable references use `input.fieldName` format (no `{{` or `}}` in the rawString)

### Examples

**Plain text (system prompts with no variables):**

Content: `"You are a helpful assistant."`

```json
"contentTokens": [
  { "type": "simpleText", "rawString": "You are a helpful assistant." }
]
```

**Single variable:**

Content: `"{{input.query}}"`

```json
"contentTokens": [
  { "type": "variable", "rawString": "input.query" }
]
```

**Multiple variables with text:**

Content: `"Analyze this document: {{input.documentText}} using criteria: {{input.criteria}}"`

```json
"contentTokens": [
  { "type": "simpleText", "rawString": "Analyze this document: " },
  { "type": "variable", "rawString": "input.documentText" },
  { "type": "simpleText", "rawString": " using criteria: " },
  { "type": "variable", "rawString": "input.criteria" }
]
```

**Adjacent variables separated by space:**

Content: `"{{input.field1}} {{input.field2}}"`

```json
"contentTokens": [
  { "type": "variable", "rawString": "input.field1" },
  { "type": "simpleText", "rawString": " " },
  { "type": "variable", "rawString": "input.field2" }
]
```

### Common Mistakes

- Forgetting to update `contentTokens` after editing `content` — causes Studio Web rendering issues
- Including `{{` or `}}` in the variable token's `rawString` — strip the delimiters
- Missing whitespace tokens between adjacent variables — each space is its own simpleText entry

## entry-points.json Schema

Defines how the agent is invoked. The `input` and `output` schemas must exactly mirror `agent.json`'s `inputSchema` and `outputSchema`.

```json
{
  "$schema": "https://cloud.uipath.com/draft/2024-12/entry-point",
  "$id": "entry-points.json",
  "entryPoints": [
    {
      "filePath": "/content/agent.json",
      "uniqueId": "<AUTO_GENERATED_UUID>",
      "type": "agent",
      "input": {
        "type": "object",
        "properties": { },
        "required": []
      },
      "output": {
        "type": "object",
        "properties": { }
      }
    }
  ]
}
```

### Sync Rule

When adding or removing input/output fields in `agent.json`, apply the same change to `entry-points.json`:

| agent.json | entry-points.json |
|-----------|-------------------|
| `inputSchema.properties.<field>` | `entryPoints[0].input.properties.<field>` |
| `inputSchema.required` | `entryPoints[0].input.required` |
| `outputSchema.properties.<field>` | `entryPoints[0].output.properties.<field>` |

The `filePath`, `uniqueId`, and `type` fields should not be modified.

## project.uiproj Schema

Project metadata file. Minimal edits needed — mainly the `Name` and `Description`.

```json
{
  "ProjectType": "Agent",
  "Name": "<AGENT_NAME>",
  "Description": null,
  "MainFile": null
}
```

| Field | Value | Editable |
|-------|-------|----------|
| `ProjectType` | Always `"Agent"` | No |
| `Name` | Agent display name | Yes |
| `Description` | Optional description | Yes |
| `MainFile` | Always `null` for low-code agents | No |

## Auto-Generated Files

These files are managed automatically. Do not create, edit, or regenerate them.

| File | Purpose | Managed By |
|------|---------|------------|
| `flow-layout.json` | UI layout positioning (zoom, agent position) | Studio Web |
| `.agent-builder/agent.json` | Expanded agent config with features/resources arrays | Studio Web |
| `.agent-builder/entry-points.json` | Mirror of entry-points.json | Studio Web |
| `.agent-builder/bindings.json` | Resource bindings (app, memorySpace) | Studio Web |
| `.project/JitCustomTypes.json` | Build metadata | Build system |

## Minimal Working Example

A simple agent with one string input and one string output (Risk Assessor pattern):

```json
{
  "version": "1.0.0",
  "settings": {
    "model": "anthropic.claude-sonnet-4-6",
    "maxTokens": 16384,
    "temperature": 0,
    "engine": "basic-v2",
    "maxIterations": 25,
    "mode": "standard"
  },
  "inputSchema": {
    "type": "object",
    "required": ["query"],
    "properties": {
      "query": {
        "type": "string",
        "description": "The user's question or request"
      }
    }
  },
  "outputSchema": {
    "type": "object",
    "properties": {
      "answer": {
        "type": "string",
        "description": "The agent's response"
      }
    }
  },
  "metadata": {
    "storageVersion": "50.0.0",
    "isConversational": false,
    "targetRuntime": "pythonAgent",
    "showProjectCreationExperience": false
  },
  "type": "lowCode",
  "projectId": "00000000-0000-0000-0000-000000000000",
  "messages": [
    {
      "role": "system",
      "content": "You are a helpful assistant. Answer the user's question clearly and concisely.",
      "contentTokens": [
        {
          "type": "simpleText",
          "rawString": "You are a helpful assistant. Answer the user's question clearly and concisely."
        }
      ]
    },
    {
      "role": "user",
      "content": "{{input.query}}",
      "contentTokens": [
        {
          "type": "variable",
          "rawString": "input.query"
        }
      ]
    }
  ]
}
```
