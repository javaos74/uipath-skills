# Plan: Create `uipath-lowcode-agents` Skill

## Context

UiPath supports two types of AI agents: **coded agents** (Python-based, covered by `uipath-coded-agents`) and **low-code agents** (declarative JSON-based, not yet covered). Low-code agents are configured entirely via JSON files — system prompts, input/output schemas, memory spaces (RAG), escalations, tools, contexts, MCPs, and evaluation sets — with no code writing required.

A new `uip lowcodeagents` CLI is being developed in parallel with `init` and `validate` subcommands. The rest of agent configuration is done by editing JSON files directly. A `uip lowcodeagents models` command is planned but not yet available.

Agents can exist **standalone** (own project in a solution) or **embedded** in a flow project (subdirectory with agent.json, features/, resources/ — no own project.uiproj, governed by parent flow).

### 2. references/agent-json-format.md (~350 lines)

Core schema reference for all project files:
- Project directory structure (from `uip lowcodeagents init`)
- **agent.json** full schema: version, settings (model, maxTokens, temperature, engine, maxIterations, mode), inputSchema, outputSchema, metadata (storageVersion, isConversational: false, targetRuntime), type: "lowCode", projectId (auto-generated, do not edit), messages[] with system/user roles and `{{input.fieldName}}` variables, contentTokens
- **contentTokens Construction Rules**: how to tokenize message content (simpleText + variable segments), sync rules, examples
- **entry-points.json** schema: $schema, entryPoints[], type: "agent", filePath: "/content/agent.json"
- **project.uiproj** schema: ProjectType: "Agent", Name, Description, MainFile: null
- **flow-layout.json**: auto-generated UI layout, do not edit
- **.agent-builder/**: auto-generated by Studio Web, do not create/edit/regenerate
- Minimal working example (Risk Assessor pattern)
- Complete example with all fields

### 3. references/cli-commands.md (~120 lines)

### 4. references/features-and-resources-guide.md (~350 lines)

**Features (Context Grounding / Memory Spaces):**
- feature.json schema: $featureType: "memorySpace", enabled, referenceKey, dynamicFewShot (threshold, resultCount, searchMode: hybrid/semantic/keyword), fieldSettings[] (field + weight)
- Directory structure: `features/<Name>/feature.json`
- Creating a memory space, connecting to Orchestrator index
- Example from BadDebtAgent (COE28595_MemorySpace)

**Resources:**

*Escalations:*
- resource.json schema: $resourceType: "escalation", actionCenter, channel (recipients, labels), inputSchema, outputSchema, outcomeMapping, appReference (appName, appVersion, resourceKey)
- Directory structure: `resources/<Name>/resource.json`
- Creating an escalation
- Example from BadDebtAgent (Contradiction Escalation)

*Tools:*
- resource.json schema: $resourceType: "tool"
- References other processes, API workflows, or other agents as callable tools
- General pattern follows escalation structure — specific schemas to be refined

*Contexts:*
- resource.json schema: $resourceType: "context"
- References ECS (Enterprise Content Service) indexes for context grounding
- General pattern follows escalation structure — specific schemas to be refined

*MCPs:*
- resource.json schema: $resourceType: "mcp"
- References MCP servers
- General pattern follows escalation structure — specific schemas to be refined

### 5. references/evaluations-guide.md (~280 lines)

- File structure: `evals/eval-sets/`, `evals/evaluators/`
- Evaluation set schema: id, name, batchSize, evaluatorRefs[], evaluations[] (test cases with inputs + evaluationCriterias)
