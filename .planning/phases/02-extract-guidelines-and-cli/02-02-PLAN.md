**6b. run-file**
From coded's uip-guide.md lines 151-182:
- Description: Run or debug a workflow file using Studio
- Two code examples inline (per decision — self-contained, no cross-reference to uia-debug-workflow.md):
  ```bash
  # Run (default — closes app on completion or error):
  uip rpa run-file --file-path "<FILE>" --project-dir "<PROJECT_DIR>" --format json

  # Debug (pauses on error — keeps app open for inspection/repair):
  uip rpa run-file --file-path "<FILE>" --project-dir "<PROJECT_DIR>" --command StartDebugging --format json
  ```
- Parameter table: --file-path (required), --command (optional — StartExecution default, StartDebugging for UI automation), --input-arguments (optional, JSON), --log-level (optional)
- List other debug commands: Stop, StepInto, StepOver, StepOut, Continue, Break, ToggleBreakpoint

- Do NOT include example JSON responses (per decision)

Assembled from RPA's cli-reference.md line 45 and coded's coding-guidelines.md line 108:

- Code example: `uip rpa get-errors --file-path "<FILE>" --format json`
- Parameter table: --file-path (optional), --skip-validation (optional, use cached errors)
- Note: `get-errors` returns cached state only — prefer `validate` when files have been changed outside Studio (forces re-analysis)
